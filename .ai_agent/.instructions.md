Instruksi ini akan membimbing AI Agent untuk generate code & provide suggestions sesuai project guidelines.

## 🎯 Project Overview

**MythosNote** adalah aplikasi AI-powered note-taking dengan fitur:
- Workspace management untuk multiple projects
- File upload & processing (PDF, Markdown, Text)
- Vector search dengan PostgreSQL + pgvector
- Chat interface dengan RAG (Retrieval Augmented Generation)
- Quick generate features (summary, table, quiz, mindmap)

**Tech Stack:**
- Backend: Django 5.x + PostgreSQL
- Frontend: Tailwind CSS + Alpine.js/Flowbite
- Queue: Redis + django-rq
- Storage: Google Cloud Storage
- AI: Gemini API / OpenAI

### Django Models

**Format:**
- Use descriptive field names
- Add help_text & verbose_name
- Add indexes untuk frequently queried fields
- Include Meta class dengan ordering & unique_together

**Example:**
```python
class Source(models.Model):
    """File yang diupload oleh user."""
    
    id = models.UUIDField(
        primary_key=True,
        default=uuid.uuid4,
        editable=False
    )
    workspace = models.ForeignKey(
        Workspace,
        on_delete=models.CASCADE,
        related_name="sources",
        help_text="Workspace yang memiliki source ini"
    )
    filename = models.CharField(
        max_length=255,
        help_text="Nama file original"
    )
    file_type = models.CharField(
        max_length=10,
        choices=[("pdf", "PDF"), ("md", "Markdown"), ("txt", "Text")],
        help_text="Tipe file"
    )
    status = models.CharField(
        max_length=20,
        default="pending",
        choices=[\n            ("pending", "Pending"),
            ("processing", "Processing"),
            ("ready", "Ready"),
            ("failed", "Failed"),
        ],
        help_text="Status pemrosesan file"
    )
    created_at = models.DateTimeField(auto_now_add=True)
    
    class Meta:
        ordering = ["-created_at"]
        indexes = [
            models.Index(fields=["workspace", "status"]),
        ]
    
    def __str__(self) -> str:
        return f"{self.filename} ({self.file_type})"
```

### Django Views & URLs

**Format:**
- Use class-based views (CBV) untuk CRUD operations
- Use function-based views (FBV) untuk custom logic
- Add proper error handling & validation
- Return appropriate HTTP status codes

**Example:**
```python
from django.views.decorators.http import require_http_methods
from django.http import JsonResponse

@require_http_methods(["POST"])
def upload_source(request):
    """Handle file upload dan queue processing."""
    try:
        file = request.FILES.get("file")
        if not file:
            return JsonResponse(
                {"error": "No file provided"},
                status=400
            )
        
        # Validation & processing
        source = Source.objects.create(
            workspace_id=request.POST.get("workspace_id"),
            filename=file.name,
            file_type=file.content_type
        )
        
        # Queue background job
        enqueue(process_source_task, source.id)
        
        return JsonResponse(
            {"source_id": str(source.id)},
            status=201
        )
    except Exception as e:
        return JsonResponse(
            {"error": str(e)},
            status=500
        )
```

## 🔍 Database Design Principles

1. **Use UUIDs** untuk primary keys (bukan auto-increment)
2. **Add timestamps** — created_at & updated_at di semua tables
3. **Use indexes** untuk frequently queried fields
4. **Normalize data** — Avoid redundancy
5. **Use pgvector** untuk embeddings (dimension: 1536 untuk OpenAI/Gemini)
6. **Add constraints** — NOT NULL, UNIQUE, FOREIGN KEY
7. **Use JSON/JSONB** untuk flexible data (misal options/config)

## 🧪 Testing Standards

**Format:**
- Write tests untuk semua models, views, & utilities
- Use pytest + pytest-django
- Aim untuk 80%+ code coverage
- Test both happy path & edge cases

**Example:**
```python
import pytest
from django.test import Client
from apps.sources.models import Source

@pytest.mark.django_db
def test_upload_source_valid_file(client):
    """Test uploading valid PDF file."""
    workspace = Workspace.objects.create(...)
    
    with open("test_file.pdf", "rb") as f:
        response = client.post(
            "/api/sources/upload/",
            {
                "file": f,
                "workspace_id": workspace.id
            }

    assert response.status_code == 201
    source = Source.objects.get(workspace=workspace)
    assert source.status == "pending"
    assert source.file_type == "pdf"

@pytest.mark.django_db
def test_upload_source_invalid_type(client):
    """Test uploading unsupported file type."""
    workspace = Workspace.objects.create(...)
    
    with open("test_file.exe", "rb") as f:
        response = client.post(
            "/api/sources/upload/",
            {"file": f, "workspace_id": workspace.id}
        )
    
    assert response.status_code == 400
```
